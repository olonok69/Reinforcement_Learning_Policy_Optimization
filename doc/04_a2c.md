# A2C (Advantage Actor-Critic)

## Core idea
A2C combines:
- **Actor**: policy $\pi_\theta(a|s)$
- **Critic**: value estimate $V_\phi(s)$

The policy uses **advantage** estimates to reduce gradient variance:

$$
A_t = G_t - V_\phi(s_t)
$$

Policy objective (simplified):

$$
\mathcal{L}_{policy} = -\mathbb{E}[\log\pi_\theta(a_t|s_t)A_t]
$$

Value objective:

$$
\mathcal{L}_{value}=\mathbb{E}[(V_\phi(s_t)-G_t)^2]
$$

Intuition:
- The actor answers: "what should I do?"
- The critic answers: "how good is this state?"
- Advantage $A_t$ centers the learning signal so updates focus on relative action quality, not raw return magnitude.

## How this repo implements it
- Module: `benchmarks/a2c.py`
- Runner: `a2c_benchmark.py`
- Shared trunk network with policy and value heads.
- Includes entropy bonus to avoid early policy collapse.

## Training flow
1. Collect one episode trajectory.
2. Compute discounted returns.
3. Compute advantages via critic baseline.
4. Optimize combined loss: policy + value - entropy.

Typical combined objective form:

$$
\mathcal{L}=\mathcal{L}_{policy}+c_v\mathcal{L}_{value}-c_e\mathcal{L}_{entropy}
$$

where $c_v$ and $c_e$ control value-fitting strength and exploration pressure.

## Strengths
- Lower variance than REINFORCE.
- Efficient, compact architecture.
- Good baseline actor-critic method.

## Limitations
- On-policy data usage (less replay efficiency).
- Stability still sensitive to coefficient tuning.

## Common failure patterns
- **Critic underfits**: noisy/biased advantages lead to unstable actor updates.
- **Entropy too low**: policy collapses to near-deterministic behavior too early.
- **Entropy too high**: policy stays too random and learning plateaus.

## Practical notes
- Tune `value_coef` and `entropy_coef` carefully.
- If policy collapses too early, raise entropy coefficient slightly.
- If value loss dominates, reduce value coefficient or improve critic capacity.
